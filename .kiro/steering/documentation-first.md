# Documentation-First Approach

## Always Check Documentation

When working with AWS services, libraries, or frameworks:
- ALWAYS search relevant documentation before making assumptions
- Use the AWS documentation search tool for AWS-specific questions
- Check library documentation for third-party packages
- Verify API signatures and parameters against official docs

## When to Search Documentation

Search documentation in these scenarios:
- Implementing new AWS service integrations
- Using unfamiliar library methods or classes
- Troubleshooting errors or unexpected behavior
- Verifying best practices and recommended patterns
- Checking for recent updates or new features
- Understanding configuration options

## Documentation Sources Priority

1. **AWS Services**: Use AWS documentation search tool
2. **Python Libraries**: Check official library documentation via Context7
3. **Framework-Specific**: FastAPI, CDK, etc. - use appropriate documentation tools
4. **General Programming**: Language-specific official documentation

## Benefits

- Ensures accurate, up-to-date information
- Reduces errors from outdated knowledge
- Discovers new features and best practices
- Provides authoritative guidance on complex topics
